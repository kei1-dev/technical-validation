ベクトルバケットの命名規則

注記

Amazon S3 Vectors は Amazon Simple Storage Service のプレビューリリースであり、変更される可能性があります。

ベクトルバケット名は、AWS リージョン内の一意性を確保するために、特定の命名規則に従う必要があります。Amazon S3 では、以下のバケット命名要件が適用されます。これらのルールに従わない場合、ベクトルバケットを作成することはできません。さらに、プログラムまたはコンソールを使用してベクトルバケットを操作するときに競合を防ぐのに役立つベストプラクティスがあります。

ベクトルバケットの命名要件

ベクトルバケットを作成するときは、以下の要件に従う必要があります。

  • ベクトルバケット名は、各 AWS リージョンごとに同じ AWS アカウントで一意である必要があります。

  • ベクトルバケット名は 3～63 文字の長さにする必要があります。

  • ベクトルバケット名は、小文字 (a-z)、数字 (0-9)、およびハイフン () のみで構成できます。

  • ベクトルバケット名は、文字または数字で開始および終了する必要があります。

命名に関するベストプラクティス

ベクトルバケットに名前を付けるときは、以下のベストプラクティスに従うことをお勧めします。

  • ベクトルデータの目的を反映するわかりやすい名前 (製品推奨、ドキュメント埋め込みなど) を使用します。

  • ログや URL に表示される可能性があるため、バケット名に機密情報を使用しないでください。

  • 管理と識別を容易にするために、名前を簡潔かつ意味のあるものにします。

これらの命名規則により、AWS マネジメントコンソール、Amazon S3 REST API、AWS CLI、および AWS SDK を介してベクトルバケットに確実にアクセスできます。
